Bring every Claude desktop account's sessions back into one list.

Log into a different Claude account and the desktop app shows an empty session list. Your conversations aren't gone — the app just scopes its session index per account. This tool reconciles that, safely, on your machine.

Why it's safe

Two different things, in two different places

The macOS desktop app keeps your transcripts separately from the small index that lists them. Only the index is scoped per account — which is the entire reason switching accounts hides your sessions, and why fixing it never risks a conversation.

WhatWhere it livesAccount-scoped?
Transcripts
the actual conversation		 ~/.claude/projects/<cwd>/<id>.jsonl No — global, shared with the CLI
Session index
pointer files: title, model, cwd, archive state		 …/Application Support/Claude/<tree>/<account>/<org>/local_<id>.json Yes — one folder per account / org

Because transcripts are global, switching accounts never loses a conversation — the UI just stops listing it. This tool only reorganizes the small pointer files. It never reads, writes, moves, or deletes anything under ~/.claude or any .jsonl.

How it works

Canonical merge, then optional link

A “location” is an <account>/<org> pair. The two steps run in order — the second is opt-in and reversible.

Step A · Merge non-destructive

Copy pointers into canonical

Pick one canonical location. Copy the pointer files that exist only in other accounts into it, and union spaces.json and scheduled-tasks.json (your routines) by id (canonical wins). Nothing is overwritten; conflicts are reported and canonical is kept.

Step B · Link opt-in · reversible

Symlink the others to canonical

With --link, back up each other account's folder to *.bak-<timestamp> and replace it with a symlink to canonical, so all accounts share one live set. Undo anytime with --revert.

The one rule that keeps it consistent: copy into canonical; symlink every other account at canonical. Never copy into a folder you then link away.

Pinned & grouped sessions live in a single global LevelDB keyed by session id, so they follow the merged sessions automatically. The tool reads it read-only to report which pins carry over — it never edits it.

Install · macOS

Three ways in — none need a prerequisite

All three give you a claude-desktop-merge command. Then just run it: it's a dry run that changes nothing until you add --apply.

1 · one-line installer

Simplest

Puts it on your PATH; installs Bun for you if you have no runtime.

2 · npm

If you have Node

Run once with npx, or install the command globally.

3 · from source

To read or hack on it

Clone and run the single TypeScript file with Bun or Node ≥ 23.6.

# downloads the tool, adds it to your PATH
curl -fsSL https://raw.githubusercontent.com/sebryu/claude-desktop-merge/main/install.sh | bash

Usage

One command; everything else is a flag

Everything is a dry run until you pass --apply. The default run discovers your accounts and prints a plan.

common invocations
$ claude-desktop-merge                                # interactive picker + plan (no changes)
$ claude-desktop-merge --canonical=A/O --sources=B/O --apply
$ claude-desktop-merge --canonical=A/O --all-others --link --apply   # quit the app first
$ claude-desktop-merge --revert                       # preview undoing a --link
FlagMeaning
--canonical=<a>/<o>Target location that receives the merged sessions.
--sources=<a>/<o>[,…]One or more source locations to merge from.
--all-othersUse every non-canonical location as a source.
--linkAlso do Step B: back up each source folder and symlink it to canonical.
--revertUndo a previous --link: restore each *.bak-<ts> backup over its symlink.
--applyExecute. Without it, the tool is a dry run.
--yes, -ySkip the confirmation prompt on --apply.
--log-file=<path>Write the run log here (default: logs/desktop-merge-<ts>.log).
--no-logDisable file logging for this run.
--help, -hShow help.

Safety

Built to be reversible and quiet

  • Dry-run by default — nothing changes without --apply.
  • Never overwrites canonical files; conflicts are reported and canonical kept.
  • Backs up every folder before symlinking (*.bak-<ts>), and --revert restores it.
  • Read-only on LevelDB; never touches ~/.claude transcripts.
  • Every run is logged to a file for an audit trail.
  • Only removes a symlink on revert — a real directory is treated as a conflict, untouched.
Quit the Claude desktop app before --link or --revert, so it doesn't recreate the folders mid-operation.
